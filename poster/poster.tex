      \pbox{Motivation}{
        Generative text-to-image models produce photorealistic images from text prompts, but cannot render a \emph{specific} subject.

        \vspace{0.4em}
        \textbf{DreamBooth} solves this: given 3--5 photos of the subject, it fine-tunes a diffusion model, allowing it to generate images of the subject in various contexts based on the textual prompt.

        \begin{center}

          \includegraphics[scale=0.45]{fine_tuning_inference.png}
          {\footnotesize Dreambooth Technique During Fine-Tuning and Inference}
        \end{center}

        \vspace{0.4em}
        \textbf{Goal.} Replicate DreamBooth on \texttt{stable-diffusion-v1-5} (SD~1.5) and verify quantitative metrics.
      }

      \pbox{Background and Key Ideas}{

        \vspace{0.4em}
        \begin{citemize}
        \item \textbf{Image labeling}: Label subjects with ``a [identifier] [class noun]'', using \textbf{rare-token identifiers} with  negligible prior meaning.
        \item \textbf{Prior-preservation loss:} Compare outputs with pre-generated class images to retain general class knowledge, preventing \textit{language drift}.
        \item \textbf{Metrics:} Ensure \textit{subject fidelity} with DINO \& CLIP-I and \textit{prompt fidelity} with CLIP-T
        \end{citemize}

        \vspace{0.4em}

        \begin{center}
          \begin{minipage}{0.9\linewidth}
            \centering
            \includegraphics[scale=1.1]{loss_diagram.png}\
        
            {\footnotesize DreamBooth Loss Components and Fine-Tuning Process}
          \end{minipage}
        \end{center}

      }

      \pbox{Implementation Details}{
        \begin{citemize}
        \item \textbf{Dataset:} Substituted Google Imagen with SD~1.5 for generated samples.
        \item \textbf{Subjects:} \texttt{dog}, \texttt{dog2}, \texttt{backpack} from the official DreamBooth dataset (google/dreambooth).
        \item \textbf{Compute:} Google Colab T4 GPU ($\sim$30--40 min per subject). Metric computation on local CPU.

        \end{citemize}
      }

    \end{column}

      \pbox{Methodology}{
        \textbf{Architecture.} SD~1.5 has three components:
        \begin{citemize}
        \item \textbf{VAE} ($\sim$83M) --- encodes/decodes images; \emph{frozen}.
        \item \textbf{UNet} ($\sim$860M) --- latent denoiser; \textbf{fine-tuned}.
        \item \textbf{CLIP encoder} ($\sim$123M) --- encodes prompts; \emph{frozen}.
        \end{citemize}

        \vspace{0.2em}
        \textbf{Training loop (per step):}
        \begin{cenumerate}
        \item Encode instance + class images to latents via frozen VAE.
        \item Encode prompts via frozen CLIP text encoder.
        \item Sample noise $\varepsilon \sim \mathcal{N}(0,I)$ and timestep $t$.
        \item Add noise: $z_t = \sqrt{\bar\alpha_t}\,z + \sqrt{1{-}\bar\alpha_t}\,\varepsilon$.
        \item Predict: $\hat\varepsilon = \mathrm{UNet}(z_t, t, \tau_\theta(P))$.
        \item Compute combined loss; backprop through UNet only.
        \end{cenumerate}

        \vspace{0.2em}
        \[
          \mathcal{L} =
          \underbrace{\|\hat\varepsilon_\mathrm{inst} - \varepsilon_\mathrm{inst}\|^2}_{\text{instance loss}}
          + \lambda
          \underbrace{\|\hat\varepsilon_\mathrm{class} - \varepsilon_\mathrm{class}\|^2}_{\text{prior preservation}}
        \]

        \vspace{0.2em}
        \textbf{Hyperparameters:}
        800 training steps, learning rate: $5\times 10^{-6}$, resolution: $512\times 512$, prior weight $\lambda=1.0$, $100$ class images, $5$ instance images, optimizer: 8-bit AdamW.

      }

      \pbox{Additional Exploration: Logo Fine-Tuning}{
        We fine-tuned on a custom \textbf{CS4782} course logo (5 augmented variants). 
        \begin{citemize}
            \item Direct generation faithfully reproduced the logo's structure and text with only minor color and number drift. \item Contextualization failed: placement prompts (shirt, mug) caused the model to render the raw token \texttt{sks} as literal glyphs rather than the learned logo.
        \end{citemize}

        \begin{citemize}
        \item \textbf{Scale to 30 subjects:} Enables direct baseline comparison.
        \item \textbf{Explore Hyperparameters:} Vary the learning rate and step count to identify the boundary between underfitting and overfitting.
        \item \textbf{Multi-subject:} Bind multiple identifiers in one fine-tune.

        \textbf{Qualitative observations:}
        \begin{citemize}
        \item Subject identity preserved across novel backgrounds and lighting.
        \item Fine tuning both the UNet and the text encoder leads to sub-optimal, overfit results
        \end{citemize}
      }

      \pbox{Conclusion}{
        We replicated the core DreamBooth algorithm on SD~1.5, writing the training
        loop from scratch and verifying each component independently.

        \vspace{0.2em}
        \textbf{Key takeaways:}
        \begin{citemize}
        \item Full UNet fine-tuning achieves strong fidelity with only 5 instance
          images and 800 steps.
        \item Metrics fall within the expected SD~1.5 range, confirming faithful
          replication.
        \item Logo fine-tuning generalizes poorly to context: direct generation
          reproduces the graphic faithfully, but placement prompts cause the model to render literal glyphs rather than the learned logo.
        \end{citemize}
      }
